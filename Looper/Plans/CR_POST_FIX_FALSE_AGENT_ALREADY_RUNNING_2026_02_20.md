# Post-Fix Code Review: False "Agent Already Running"

**Date:** 2026-02-20
**Reviewer:** Antigravity AI
**Target:** `Looper/StartLoopsInWT.py` (Local Fix)

## 1. Обзор изменений (Overview of changes)

В соответствии с RCA и согласованным планом лечения, были внесены следующие изменения в файл `StartLoopsInWT.py`:

1. **WMI Исключение хостов терминала:**
   * В функции `get_process_command_lines()` WMI-запрос обновлен: запрашивается не только `CommandLine`, но и `Name`. 
   * Добавлена логика исключения: процессы с именем `wt.exe` или содержащие `windowsterminal` игнорируются. Это устраняет **первичную причину** проблемы — zombie-кеширование путей в командной строке `wt.exe`, которое навечно блокировало запуск упавших воркеров после первой попытки (bootstrap fail).

2. **Точный парсинг аргументов:**
   * Добавлена функция `extract_cmd_arg`, которая детерминированно извлекает значение конкретного флага (например, `--project-root` и `--agent-path`) из командной строки.
   * Парсер учитывает кавычки (одинарные/двойные), экранирования (типа `\"`), и отсутствие кавычек.

3. **Строгая валидация (Strict Match):**
   * Из `StartLoopsInWT.py` удалена слабая эвристическая функция `contains_path_token`, которая выдавала ложно-положительные срабатывания на совпадениях подстрок (например `C:\CorrisBot` внутри `C:\_RUN_CorrisBot`).
   * В функции `test_agent_already_running()` теперь явно достаются аргументы `cmd_project` и `cmd_agent`, нормализуются слэши и регистр, и проводится строгое сравнение (`==`). 

## 2. Анализ Регрессий (Regression Analysis)

Изменения были проверены (включая синтаксическую валидацию `py -m py_compile`) и не вносят регрессий:
* **True Positives:** Реальные процессы (`cmd.exe /c CodexLoop.bat ...` или `python.exe codex_prompt_fileloop.py ...`) имеют полное тело вызова с флагами `--project-root` и `--agent-path`. Они по-прежнему будут распознаны, и защита от второго запуска того же воркера будет корректно работать.
* **False Positives:** Терминальный хост больше не обманывает проверку. Другие проекты по совпадению подстрок не мешают друг другу.
* **Синтаксическая совместимость:** Использование строковых манипуляций вместо сложных RegEx устраняет риск зависания (hang) Python-движка, который наблюдался при работе мощных RegEx выражений над системными `CommandLine` строками без кавычек.

**✅ Fix is solid and ready.** 

Код успешно применяет fail-closed подход к определению "already running" состояний, строго основываясь на реальных параметрах запуска. Фикс полностью решает задачу *ложного `Agent already running` при bootstrap Worker*. Угроза сломать routing contract исключена, так как логика маршрутизации вообще не затронута.
